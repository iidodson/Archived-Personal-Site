The Asset Toolkit is a centralized code based style guide that keeps styles and components consistent across The User Testing product. From user research to launch, I created a mobile-first website that gave developers, designers, and product managers direct access to these design components.

View Prototype

Objectives

My objectives for this project were to:

1. Conduct market research on style guides
2. Administer usability tests to gauge visual and contextual successes and pain points for users
3. Design and iterate on a prototype that represents the goals of the frontend team and resolves major pain points for the user.
4. Code the prototype

Process

I met with the frontend team gain a deeper understanding of the team culture, existing brand, and their expectations for the toolkit. I also met with product managers and designers to determine secondary use cases.

I created use cases for each user group to aid us in determining the scope of the project.

Frontend Developers need to:
1. View code for each design component
2. Know where to find the code internally
Designers need to:
1. See visual representation of each style and component
2. Find specifications of each component
Product managers:
1. Read the use case for each component

User Tests

I performed a competitor analysis of style guides in the frontend development community then tested them against each other using the User Testing platform.

The goal of these user tests was to understand:

1. The type of content the user expected to find in a typical style guide
2. The visual expectations users had for a style guide
2. How the company's needs compared to the industry standards
